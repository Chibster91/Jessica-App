Design notes

  • No-account framing. The screen header subtitle says "Stored on this device" — no sign-in copy anywhere. Avatar is a single initial (or a placeholder glyph) drawn from the user's name in localStorage.
  • Read-only KV rows. Hairline-separated, key in --fg-2, value in mono so numbers right-align cleanly. Same row pattern as Library/Profile rows already in the kit.
  • Targets card. BMR / TDEE / Recommended calorie target sit in the same stat-card style used on the Weight Current screen, then macros below in protein/carbs/fat colors. Lifts straight from the existing macro palette.
  • Wizard. Five steps:
    1. Identity — Name, Age, Sex
    2. Body — Height, Current weight
    3. Activity — Sedentary / Light / Moderate / Very active / Athlete
    4. Plan — Goal (lose/maintain/gain), Pace (slow/moderate/aggressive), Goal weight, Goal date (shown)
    5. Review — Live BMR / TDEE / Target / Macro split, with toggleable macro split
    Progress bar at the top, primary purple Next, ghost Back. Live calculation block updates as you change inputs.
  • Segmented option groups. Same pill-card pattern across Sex / Activity / Goal / Pace — each option shows its label plus a one-line definition (e.g. "−1.0 lb/wk") so users don't have to guess what "moderate" means.
  • Delete-all confirmation. Three guards:
    1. Modal must be opened intentionally (it sits inside Profile → Data, not on a button).
    2. A summary block tells the user exactly what will be erased (counts pulled from localStorage).
    3. The user must type DELETE in mono before the red destructive button enables.
    The destructive button uses --danger-bg/--danger-fg from the system, not invented red.
